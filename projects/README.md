# Projects

This folder is the active working area of the vault.

If `wiki/` is the durable memory, `projects/` is the place for live research, open questions, and evolving synthesis.

## What belongs here

Use `projects/` for:

- project briefs
- open questions
- replication work
- validation notes
- comparison notes
- temporary synthesis
- decisions and next steps

This is where the agent and human can work through uncertain material before deciding what deserves a stable place in `wiki/`.

## Recommended project shape

Each serious project should eventually answer:

- What are we trying to understand?
- Why does it matter?
- What sources are relevant?
- What evidence exists?
- What remains unresolved?
- What should be promoted to `wiki/`?

Typical files inside a project folder may include:

- `brief.md`
- `questions.md`
- `plan.md`
- `findings.md`
- `decision.md`

Not every project needs all of them, but the reasoning should remain inspectable.

## Relationship to other folders

- `raw/` provides source material
- `catalog/` provides mirrored data understanding
- `artifacts/` provides bounded evidence outputs
- `wiki/` receives stable conclusions

Projects should link outward rather than duplicate large source content.

## Lifecycle rule

Projects are allowed to be messy while work is active.

But when something becomes durable, the agent should crystallize it:

- stable dataset understanding goes to `wiki/`
- stable method comparisons go to `wiki/`
- stable project evidence should be linked from `artifacts/`

The point of `projects/` is not permanence. It is controlled evolution.
